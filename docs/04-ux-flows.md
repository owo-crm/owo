# UX Flows

Status: Draft  
Owner: Product + Frontend  
Last updated: 2026-04-02

## Surfaces
- Telegram Mini App
- Web App

## Shared flow set (must match across surfaces)
- Sign in and business selection
- Lead intake and owner assignment
- First-contact execution
- Task follow-up and completion
- Lead status progression

## Surface-specific constraints
### Telegram
- Compact layouts, low interaction depth.
- Fast action shortcuts.
- Limited screen real estate is default.

### Web
- Denser data views.
- Multi-column and bulk operations.
- Better support for deep editing.

## UX parity rule
Business outcome parity is mandatory. UI can differ, core behavior cannot.

## Flow template
- Trigger
- Actor
- Steps
- System reactions
- Success condition
- Failure and recovery path
